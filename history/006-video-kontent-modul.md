# 006 — Видео-контент: модуль коротких видео (Reels-style)

Дата: 2026-07-12

## Контекст

После завершения всех 13 пунктов дорожной карты ENGINE.md §9 (Orders, Reviews, Seller dashboard, Search, Real-time, CMS+Marketing, Moderation, Finance, Blogger, Notifications) была проведена полная проверка фронтенда вживую — все страницы работали корректно на реальном backend.

Пользователь спросил, что осталось по планам. Единственным пробелом, явно требуемым PLAN.md, но не реализованным нигде в backend, оказался видео-контент (короткие видео в стиле TikTok/Reels у покупательской ленты `/video`, у блогера, и раздел модерации видео в админке) — до этого момента везде существовал только mock (`videos`, `comments`, `bloggerVideos` массивы), с явными комментариями в коде "Intentionally still mock: no video-content entity exists on the backend yet".

Через `AskUserQuestion` был согласован масштаб v1: **только ядро** — загрузка, лента, лайки, комментарии, тег товара, модерация. Явно отложены: привязка видео к live-эфиру (кнопка "Смотреть эфир"), фасет поиска "по видео", вложение видео в чаты.

## Решение

Архитектура максимально переиспользует существующие паттерны вместо изобретения новых:

- **Media presign-flow** (уже существовавший для товарных изображений) переиспользован без изменений — видео-модуль просто хранит `publicUrl` как `videoUrl`, kind: "video" уже был предусмотрен в `MediaKind`.
- **Один `productId?: number` на `VideoEntity`**, а не массив `productIds[]` (как у `StreamEntity`) — потому что все mock-формы видео всегда содержат ровно один тег товара.
- **`VideoCommentEntity`** смоделирован по образцу `ChatMessageEntity`, с добавлением `likesCount` (которого нет у чат-сообщений), так как mock `comments` хранил лайки на комментарий. Реальный лайк-трекинг комментариев в v1 не реализован — поле есть для соответствия форме, но лайкать сами комментарии нельзя.
- **Generic `ReportEntity`/Moderation** расширен добавлением `"video"` в объединение `targetType` вместо отдельной модерационной сущности — точечные правки в существующих if-цепочках `assertTargetExists`/`applyAction`.
- **`NotificationType`** расширен новым значением `"social"` (не переиспользован `"review"`) — точное соответствие категории "Соціальні" из исходного мока уведомлений.
- Самоуведомление подавляется явной проверкой `video.authorId !== userId` перед эмитом `video.liked`/`video.commented`.
- Роли: создавать/удалять видео могут только `seller`/`blogger`; лайкать и комментировать — любой авторизованный пользователь; владелец видео проверяется отдельно от роли при удалении.

## Изменения

**Backend** (`apps/api/src/videos/`):
- `VideoEntity`, `VideoLikeEntity` (уникальный индекс `videoId+userId`), `VideoCommentEntity`
- `VideosService`/`VideosController`/`VideosModule` — лента (`GET /videos`), мои видео (`GET /videos/my`), одно видео, регистрация просмотра, create/delete, лайк-toggle (атомарный счётчик через транзакцию), комментарии
- Миграция `1783858963272-AddVideos.ts` (со стандартной для этой сессии ручной вырезкой спурьезного searchVector-diff)
- Расширены существующие модули: `moderation` (`ReportTargetType` + `"video"`, ветки в `assertTargetExists`/`applyAction` для скрытия видео), `notifications` (`NotificationType` + `"social"`, новый `VideoNotificationsListener`)
- `VideosModule` зарегистрирован в `AppModule`

**Frontend**:
- `packages/shared/src/index.ts` — типы `Video`, `VideoComment`, `VideoListResponse`, `ReportTargetType` + `"video"`
- `apps/web/lib/data/useVideos.ts` — новый хук-обёртка (лента, мои видео, комментарии, like-toggle, create/delete, `videoThumbnailUrl`)
- `apps/web/app/(buyer)/video/page.tsx` — переведён с mock на реальный API, лайки/комментарии через реальный UI
- `apps/web/app/(admin)/admin/moderation/videos/page.tsx` — подключён к `/reports?targetType=video`
- `apps/web/app/(blogger)/blogger/videos/page.tsx` — подключён к `GET /videos/my`
- Новая страница `apps/web/app/(seller)/seller/videos/page.tsx` — загрузка видео через presign-flow, новый пункт "Відео" в `SellerNav.tsx`
- Попутно исправлен предсуществующий баг в `AdminNav.tsx`: бейджи "Модерація відео"/"Модерація товарів" брались из мок-массивов вместо реальных `/reports` — заменены на счётчики через `usePendingReports()` с фильтром по `targetType` (бейдж "Модерація коментарів" сознательно оставлен на моке — та страница целиком мок по отдельной, ранее зафиксированной причине: комментарии к контенту — отдельная нереализованная фича, не то же самое, что чат)

## Результат проверки

Полное живое тестирование backend через прямые HTTP-запросы (создание видео, лента, лайк-toggle дважды, комментарий, просмотр, репорт → одобрение → видео скрыто и пропадает из ленты, проверка уведомлений автору с корректным подавлением самоуведомления, проверка ролевых 403/401-границ) — все сценарии прошли корректно. Тестовые данные полностью очищены после каждого прогона.

Дополнительно проведена проверка через Playwright в реальном браузере: все 4 страницы (seller/videos, buyer video, blogger/videos, admin/moderation/videos) рендерятся без ошибок консоли; реальный клик по лайку и отправка комментария через UI (не API напрямую) корректно обновили счётчики на экране и в базе данных. S3 не настроен в этом окружении, поэтому фактическая загрузка файла через presign не протестирована вживую — только структура запроса/ответа проверена по коду.
